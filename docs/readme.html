For each File — Package Overview

This document provides a brief overview of the For each File package.

You can find announcements for new releases at http://freshmeat.net/projects/ff/ and additional resources at http://sourceforge.net/tracker/?group_id=106777.

  1. Limitations

    • Target names containing new-line characters are not handled properly if they are specified via the --pipe option.
    • The translations may be incoherent at times because most of them were generated by software.

  2. Requirements

    It is recommended that the latest available versions of the following software be used, as newer releases tend to have less defects. Also, the paths to the software below must be listed before those of their native counterparts in the user's $PATH variable. For example, on a non GNU/Linux system: PATH=/gnu/bin:/bin

    Required Software

    • GNU BASH (Shell)
      • Version 2.05b has been tested successfully with this package.
      • The shell must be accessible via: /bin/bash. If this is not possible, then the user can adjust the line #!/bin/bash to reflect the path on their system in each of the shell scripts provided by this package: for f in ./f* ; do sed -e "1s;/bin/bash;${0};" "$f" > ./$$ && mv -f ./$$ "$f" ; done ; rm -f ./$$
    • GNU coreutils (Core Utilities)
      • Version 5.2.1 has been tested successfully with this package.
    • GNU sed (Stream Editor)
      • Version 4.0.8 has been tested successfully with this package.

    Recommended Software (Optional)

    • GNU termutils (Terminal Utilities)
      • Version 2.0 has been tested successfully with this package.
    • GNU less (Pager)
      • Version 382 has been tested successfully with this package.
      • If you do not want this software to be invoked by default, then make sure that the $PAGER environment variable is set.

  3. Package Tree

    The following diagram shows the hierarchical structure of this package. All paths shown in the diagram are relative to the directory that contains the package. Also, subordinate files have been omitted from the diagram for brevity.

    • ./fl — The For each Line tool.

    • ./libs/ — Contains libraries for use by shell scripts.
    • ./scripts/ — Contains example scripts that are usable by the tools of this package.

  4. Installation

    1. Make the tools in the package directory executable: chmod +x ./f*
    2. Move or copy the package directory to a location of your choice: mv . /foo/bar
    3. (Optional) Create symbolic links to the tools in a directory that is listed in your $PATH variable, such as $HOME/bin/: cp -l /foo/bar/f* "$HOME/bin/"

  5. Example Scripts

    In the ./scripts/ directory there are several example scripts that can be invoked with the tools of this package. These scripts are named such that their names begin with the name of the tool for which they were designed to be invoked by. Note that scripts written to be invoked by one tool may not produce correct results if invoked with another.

    To see how the preset variables and control functions are used, invoke the example scripts named ./scripts/ff.info and ./scripts/fl.info: ./ff -f ./scripts/ff.info ./* | ${PAGER:-less}
    ./ff -rf ./scripts/ff.info . | ${PAGER:-less}
    ./ff -f ./scripts/ff.info /bin/* | ${PAGER:-less}
    ./ff -rf ./scripts/ff.info /bin | ${PAGER:-less}
    ./fl -f ./scripts/fl.info ./docs/gpl.txt | ${PAGER:-less}

    Also, the first few lines of each example script contains a header that describes its purpose. To view these descriptions, you can execute: head ./scripts/* | ${PAGER:-less}

  7. Translations

    Feel free to contribute translations for new languages and revise the existing GNU gettext compatible translation catalogs located in ./i18n/. New and updated translation catalogs will be made available at http://ff-bash.sourceforge.net/i18n/ as they are contributed.

    Instructions for writing new translations

    1. Determine the ISO-639 code of the language you wish to write translations for.
    2. Make a copy of the translation catalog template file: cd ./i18n/
      cp foo.pot foo.ISO-639.po
    3. Write your translations in the copied file:

      msgid "Text that has not been translated."
      msgstr "Your translation."
