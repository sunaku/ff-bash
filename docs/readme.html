Installation

You can either run the installation script by executing $ ./install or perform a manual installation as follows:

  1. Simply move or copy the directory containing this package to a location of your choice.
  2. For easy access, you can create symbolic links, to the tools in this package, in a directory that is listed in your $PATH variable, such as $HOME/bin/.

Translation

Translation is performed dynamically through the use of gettext translation catalogs. These catalogs are located in the i18n/ directory.

Feel free to contribute localizations for new languages and revise the existing translation catalogs. New and updated translation catalogs will be made available at http://people.ucsc.edu/~skurapat/pub/projects/ff/i18n/ as they are contributed.

Writing new translations

First, we must identify the ISO-639 language code for the language we wish to write new translations in. Second, we must make a copy of the gettext translation catalog template file with the appropriate file name. Finally, we can begin translating by editing the copy.

For example, consider this scenario: we wish to translate some documentation for the language whose ISO-639 language code is XX. The translation catalog template for documentation is named foo.pot

  1. We make a copy of the template file.
    $ cp foo.pot foo.XX.po
  2. We edit the lines that begin with "msgstr" in the copy.

Example Scripts

First try executing (see example below) the scripts named ./scripts/ff.vars and ./scripts/fl.vars, to see how the preset variables behave. You will notice that the preset variables are consistent between recursive and non-recursive modes.

  • $ ./ff * -f ./scripts/ff.vars | more
  • $ ./ff . -rf ./scripts/ff.vars | more
  • $ ./ff /bin/* -f ./scripts/ff.vars | more
  • $ ./ff /bin/ -rf ./scripts/ff.vars | more
  • $ ./fl ./docs/gpl.txt -f ./scripts/fl.vars | more

You can also try executing the other example scripts in the scripts/ directory. Also, the first few lines of each script file describes its purpose, thus you can execute $ head ./scripts/* | more to see these descriptions.
