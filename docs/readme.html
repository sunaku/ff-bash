For each File — Package Overview

This document provides a brief overview of the For each File package.

You can find announcements for new releases at http://freshmeat.net/projects/ff/ and additional resources at http://sourceforge.net/tracker/?group_id=106777.

  1. Limitations

    • File names containing new-line characters are not handled properly.

  2. Requirements

    It is recommended that the latest available versions of the following software be used.

    • GNU BASH Shell — This package has been tested with version 2.05b.
    • GNU Core Utilities — This package has been tested with version 5.2.1.

  3. Package Tree

    The following diagram shows the hierarchical structure of this package. All paths shown in the diagram are relative to the directory that contains the package. Also, subordinate files have been omitted from the diagram for brevity.

    • ./ff — The For each File tool.
    • ./fl — The For each Line tool.
    • ./docs/ — Contains documentation for this package.
      • readme.html — This document.
      • changes.html — A list of changes between versions of this package.
      • faq.html — Answers to frequently asked questions.
      • gpl.txt — A copy of the GNU General Public License: Version 2.
    • ./i18n/ — Contains internationalization resources.
    • ./libs/ — Contains libraries for shell scripts.
    • ./scripts/ — Contains example scripts usable by the tools of this package.

  4. Installation

    1. Make the tools in the package directory executable: $ chmod +x ./f[fl]
    2. Move or copy the package directory to a location of your choice: $ mv . your_location
    3. (Optional) Create symbolic links to the tools in a directory that is listed in your $PATH variable, such as $HOME/bin/: $ cp -l your_location/f[fl] $HOME/bin/

  5. Example Scripts

    In the ./scripts/ directory there are several example scripts that can be invoked with the tools of this package. Note that scripts written to be invoked by one tool may not produce correct results if invoked with another.

    To see how the preset variables and control functions are used, invoke the example scripts named ./scripts/ff.info and ./scripts/fl.info: $ ./ff * -f ./scripts/ff.info | less
    $ ./ff . -rf ./scripts/ff.info | less
    $ ./ff /bin/* -f ./scripts/ff.info | less
    $ ./ff /bin/ -rf ./scripts/ff.info | less
    $ ./fl ./docs/gpl.txt -f ./scripts/fl.info | less

    Also, the first few lines of each example script contains a header that describes its purpose. To view these descriptions, you can execute: $ head ./scripts/* | less

  6. Support and Feedback

    You can report defects, request for support and features, submit patches, ask questions, and discuss related topics at https://sourceforge.net/tracker/?group_id=106777.

  7. Translations

    Feel free to contribute translations for new languages and revise the existing GNU gettext compatible translation catalogs located in ./i18n/. New and updated translation catalogs will be made available at http://ff-bash.sourceforge.net/i18n/ as they are contributed.

    Instructions for writing new translations

    1. Determine the ISO-639 code of the language you wish to write translations for.
    2. Make a copy of the translation catalog template file: $ cd ./i18n/
      $ cp catalog_name.pot catalog_name.ISO-639_code.po
    3. Write your translations in the copy:
      msgid "Untranslated text."
      msgstr "Your translation."
